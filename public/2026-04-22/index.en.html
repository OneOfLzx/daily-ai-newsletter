How The Heck Does Shazam Work? (An Interactive Exploration) (15 minute read)

Tech Breakdown

How everyday tech actually works — April 20, 2026

Curious how Shazam identifies songs in seconds? We break down its clever math/tricks.

How Shazam Works

Shazam skips melodies/lyrics—here’s its 4-step trick:

  1. Sound → Spectrogram: Your phone turns audio into a waveform, then runs FFT (Fast Fourier Transform) on small slices. FFT splits slices into frequencies, making a spectrogram (time=horizontal, frequency=vertical, brightness=loudness).

  2. Keep only peaks: It discards 99% of the spectrogram, keeping only the loudest peaks (a "constellation map"). This filters noise and speeds up matching.

  3. Hash peak pairs: A single peak isn’t unique—but pairs (two frequencies + time gap) are. Each pair becomes a hash (short unique string).

  4. Inverted index lookup: Instead of searching songs, Shazam looks up each hash to find matching songs. It checks if time gaps between hashes align—if so, that’s your song.

Modern Twist

Newer tools (Apple’s on-device, Google Pixel Now Playing) run locally (faster, private) but use smaller curated databases (tradeoff: slower new-song updates).

Quick Note

Shazam’s secret: Ignore most data to focus on unique landmarks. For deeper dive, check Avery Wang’s 2003 paper.

🛡️ Protect .git from Malicious Agents

When using AI tools like Claude Code in containers, mounting your project gives the agent access to .git—a hidden risk.

Why? .git/config includes diff/merge tools that trigger arbitrary code execution when you run git diff/show on the host. The agent could lie about changes or run code directly.

The simple fix:

  1. Mount your project read/write in the container (e.g., /src).
  2. Overlay .git as read-only (e.g., /src/.git).

This lets the agent view history/edit code, but you safely review changes before committing.

Key caveats:

  • Doesn’t block subdirectory repos (agent-created ones risk host code execution).
  • Disable git hooks (agent controls their contents).
  • AppArmor (kernel-level) works but isn’t supported in Docker Desktop.

Final takeaway: Use this only if the agent has internet access/untrusted input—skip if you tightly control both.

Modernizing the Facebook Groups Search to Unlock the Power of Community Knowledge

Meta has modernized Facebook Groups Search to resolve three core user friction points—discovery (keyword mismatch between queries and community content), consumption (effort to find consensus in long threads), and validation (difficulty accessing trusted community expertise)—with a hybrid lexical-semantic architecture.

The solution integrates:

  1. Parallel retrieval: Combines Facebook’s Unicorn inverted index (exact term matches) with a 12-layer semantic retriever (encodes queries/posts into vectors for conceptual similarity via Faiss).
  2. MTML ranking: A multi-task model optimizing for clicks, shares, and comments to surface relevant, engaging results.
  3. Automated evaluation: Uses Llama 3 to grade relevance (including "somewhat relevant" for domain alignment) at scale, eliminating human labeling bottlenecks.

Early results show improved search engagement with no increased error rates. Future plans include integrating LLMs directly into ranking and adaptive retrieval that adjusts to query complexity.

The work is detailed in the paper Modernizing Facebook Scoped Search: Keyword and Embedding Hybrid Retrieval with LLM Evaluation.
