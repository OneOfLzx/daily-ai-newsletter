thedotmack/claude-mem

claude-mem 仓库概述

claude-mem 是面向 Claude、Gemini 等 AI 助手的持久化记忆系统，可记录工作会话、提取可操作观察点，并为后续交互提供上下文，解决 AI 会话间的上下文丢失问题。

核心目标

保留并检索工具使用、文件编辑、决策等相关工作上下文，提升 AI 助手工作效率，减少重复工作，实现长期知识整合。

核心功能

  1. 自动观察点捕获：

    • 跟踪工具执行、文件读写、Shell 命令及会话事件。
    • 通过 Gemini、Claude、OpenRouter 等 AI 服务商提取标题、事实、概念等结构化元数据。

  2. 高效 Token 三层检索：

    • 检索：通过 MCP 工具查找相关观察点。
    • 时间线：查看结果的时间顺序上下文。
    • 详情获取：仅在需要时加载完整观察点内容。

  3. IDE 与工具集成：

    • Cursor：Cursor IDE 会话的持久化记忆。
    • Gemini CLI：集成 Google Gemini API。
    • OpenRouter：通过统一 API 访问 100+ AI 模型。
    • Claude Desktop：Claude 桌面端的记忆检索 MCP 工具。

  4. 知识管理：

    • 知识代理：从历史观察点构建可查询的语料库。
    • 目录上下文：自动生成 CLAUDE.md 文件，提供目录级上下文。
    • 私有标签：使用 <private> 标签排除敏感内容存储。

  5. 稳健架构：

    • 工作服务：基于 Express.js + Bun 的 HTTP API，负责处理与检索。
    • 数据库：SQLite（FTS5 全文检索）+ Chroma（向量检索）。
    • 手动恢复：崩溃或重启后修复卡住的观察点。

技术亮点

  • 渐进式披露：优先加载元数据而非完整内容，减少 Token 消耗。
  • 跨服务商支持：在 Claude、Gemini、OpenRouter 间无缝切换。
  • 幂等操作：通过事务处理避免重复观察点。

快速开始

  1. 通过 npx claude-mem install 或插件市场安装。
  2. 在 ~/.claude-mem/settings.json 中配置 AI 服务商（如 Gemini、OpenRouter）。
  3. 使用 MCP 工具（如 search、timeline）检索记忆。

更多详情请参考 官方文档。

核心奖励机制

  • RLVR（可验证奖励）：针对客观任务（数学、代码生成）的确定性代码。示例：情感分析中，将输出与真实值对比，格式正确可获部分分数。
  • RLAIF（AI 反馈）：通过 Amazon Bedrock 调用 AI 模型作为主观任务（写作、语气）的评判者。示例：Bedrock 模型以 0.0 温度（确保一致性）对比输出的语义相似度。

Lambda 优化技巧

  • 配置调整：复杂评估场景下，超时设为 60 秒、内存调至 512MB。
  • 冷启动优化：包体积≤50MB，全局初始化 Bedrock 客户端，缓存评分标准。
  • 错误处理：瞬时错误用指数退避重试，日志输出至 CloudWatch（跟踪零奖励、评估缓慢等问题）。
  • 权限控制：为 Bedrock 访问配置最小权限 IAM 角色。
  • 吞吐量优化：调整 SageMaker 批次大小或 HyperPod 工作节点副本数；RLAIF 选轻量评判模型。

奖励函数最佳实践

  • 平滑奖励空间：对渐进式进步给部分分数（避免二元通过/失败）。
  • 多维度评分：评估正确性、安全性、格式及简洁性，防止奖励黑客行为。
  • 可验证性：客观任务用代码实现，主观任务需验证 LLM 评判者可靠性。

优势

Lambda 无服务器架构具备高扩展性（每秒 10~400+ 次评估）、成本效益（按需付费）及训练加速能力，无需大量标注数据集即可落地 RFT。

该方法帮助企业自定义 Nova 模型，实现精准的领域特定行为。
