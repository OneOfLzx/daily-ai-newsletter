thedotmack/claude-mem

claude-mem Repository Summary

claude-mem is a persistent memory system for AI assistants (Claude, Gemini, etc.) that captures work sessions, extracts actionable observations, and provides context for future interactions—solving the problem of context loss across AI sessions.

Core Purpose

Retain and retrieve relevant work context (tool usage, file edits, decisions) to enhance AI assistant productivity, reduce redundant work, and enable long-term knowledge synthesis.

Key Features

  1. Automatic Observation Capture:

    • Tracks tool executions, file reads/writes, shell commands, and session events.
    • Extracts structured metadata (titles, facts, concepts) via AI providers (Gemini, Claude, OpenRouter).

  2. Token-Efficient 3-Layer Search:

    • Search: Find relevant observations via MCP tools.
    • Timeline: Get chronological context for results.
    • Get Details: Fetch full observation content only when needed.

  3. IDE & Tool Integration:

    • Cursor: Persistent memory for Cursor IDE sessions.
    • Gemini CLI: Integrate with Google Gemini API.
    • OpenRouter: Access 100+ AI models via unified API.
    • Claude Desktop: MCP tools for memory search in Claude Desktop.

  4. Knowledge Management:

    • Knowledge Agents: Create queryable corpora from past observations.
    • Folder Context: Auto-generate CLAUDE.md files for directory-level context.
    • Private Tags: Exclude sensitive content from storage with <private> tags.

  5. Robust Architecture:

    • Worker Service: HTTP API (Express.js + Bun) for processing and retrieval.
    • Database: SQLite (FTS5 for full-text search) + Chroma (vector search).
    • Manual Recovery: Fix stuck observations after crashes/restarts.

Technical Highlights

  • Progressive Disclosure: Minimizes token usage by prioritizing metadata over full content.
  • Cross-Provider Support: Seamlessly switch between Claude, Gemini, and OpenRouter.
  • Idempotent Operations: Prevents duplicate observations via transactional processing.

Getting Started

  1. Install via npx claude-mem install or plugin marketplace.
  2. Configure AI provider (Gemini, OpenRouter, etc.) in ~/.claude-mem/settings.json.
  3. Use MCP tools (e.g., search, timeline) to retrieve memory.

For more details, see the official docs.

How to build effective reward functions with AWS Lambda for Amazon Nova model customization

Summary: AWS Lambda for Amazon Nova Reward Functions

This blog details how to use AWS Lambda to build reward functions for customizing Amazon Nova models via Reinforcement Fine-Tuning (RFT)—a technique that teaches models desired behaviors through iterative feedback (vs. Supervised Fine-Tuning’s reliance on labeled examples).

Key Reward Mechanisms

  • RLVR (Verifiable Rewards): Deterministic code for objective tasks (math, code generation). Example: Sentiment analysis that checks responses against ground truth with partial credit for correct formatting.
  • RLAIF (AI Feedback): AI models (via Amazon Bedrock) as judges for subjective tasks (writing, tone). Example: Bedrock models comparing responses for semantic similarity (temperature 0.0 for consistency).

Lambda Optimization Tips

  • Configuration: Increase timeout to 60s and memory to 512MB for complex evaluations.
  • Cold Starts: Keep packages <50MB, initialize Bedrock clients globally, cache rubrics.
  • Error Handling: Retry transient errors (exponential backoff), log to CloudWatch (track zero rewards, slow evaluations).
  • Permissions: Least-privilege IAM roles for Bedrock access.
  • Throughput: Tune batch sizes (SageMaker) or worker replicas (HyperPod); choose minimal judge models for RLAIF.

Best Practices for Reward Functions

  • Smooth Landscapes: Partial credit for incremental progress (avoid binary pass/fail).
  • Multi-Dimensional Scoring: Evaluate correctness, safety, format, and conciseness to prevent reward hacking.
  • Verifiability: Use code for objective tasks; validate LLM judges for subjective ones.

Benefits

Lambda’s serverless architecture delivers scalability (10–400+ evaluations/sec), cost-effectiveness (pay-per-use), and faster training—making RFT accessible without massive labeled datasets.

This approach empowers organizations to customize Nova models for precise, domain-specific behaviors.
