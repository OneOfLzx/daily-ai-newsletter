Scrapling: A Modern Web Scraping Framework for Python

Overview

Scrapling is a Python web scraping framework designed to handle modern websites (dynamic content, anti-bot protections) with minimal code. It combines fetching, parsing, and crawling into a unified toolkit, offering features like adaptive scraping (survives site changes) and async crawling.

Core Components

1. Fetchers

Different fetchers for diverse use cases:

  • Fetcher: Fast static HTTP requests (uses curl_cffi for speed).
  • DynamicFetcher: Playwright-based for dynamic JS-rendered sites.
  • StealthyFetcher: Anti-bot bypass (Cloudflare, Turnstile, CAPTCHAs) with browser fingerprinting.

2. Parsers

  • Selector: Core parsing engine supporting CSS/XPath selectors, regex, text matching, and adaptive scraping.
  • Adaptive Scraping: Saves element properties to find them even if the site’s structure changes (e.g., new class names).

3. Spiders

Async crawling framework (inspired by Scrapy) for multi-page crawls:

  • Pause/Resume: Checkpointing for long crawls.
  • Concurrency Control: Limit requests per domain/second.
  • Sessions: Persistent fetchers (e.g., FetcherSession for cookies, AsyncStealthySession for anti-bot).
  • Proxy Rotation: ProxyRotator to avoid rate limits.

4. CLI Tools

  • scrapling extract: No-code scraping (e.g., scrapling extract get "https://example.com" output.txt).
  • Interactive Shell: IPython-based for rapid prototyping (e.g., get("https://news.ycombinator.com"); page.css("a")[:5]).

Key Features

  • Adaptive Scraping: Reduces maintenance from site changes.
  • Anti-Bot Bypass: StealthyFetcher handles Cloudflare, CAPTCHAs, and browser fingerprinting.
  • Async Crawling: Efficiently crawl thousands of pages with concurrency.
  • Flexible Selection: CSS/XPath, regex, text matching, and "similar element" finding.
  • Fast Parsing: Benchmarks show 10–100x faster than BeautifulSoup for large HTML.

Examples

Basic Fetch & Parse

from scrapling import Fetcher

Adaptive Scraping

Spider Crawl

from scrapling.spiders import Spider, Response

class QuotesSpider(Spider):
    name = "quotes"
    start_urls = ["https://quotes.toscrape.com"]

    async def parse(self, response: Response):
        for quote in response.css("div.quote"):
            yield {
                "text": quote.css("span.text::text").get(""),
                "author": quote.css("small.author::text").get(""),
            }
        # Follow next page
        next_page = response.css("a.next::attr(href)").get()
        if next_page:
            yield response.follow(next_page)

# Run spider
spider = QuotesSpider()
spider.start()

Kronos: Detailed Codebase Explanation

Kronos is an open-source foundation model for financial time series forecasting (K-line data: OHLCV). It uses a hierarchical tokenization approach to convert continuous market data into discrete tokens, then a Transformer-based model to generate autoregressive predictions. Below is a breakdown of its core components:

1. Core Components & Architecture

A. Hierarchical Tokenizer (KronosTokenizer)

Converts raw OHLCV data into discrete tokens using Binary Spherical Quantization (BSQuantizer). Key features:

  • Encoder-Decoder Transformer: Processes input data to learn meaningful representations before quantization.
  • BSQuantizer: Splits tokens into two parts (s1_bits + s2_bits) for hierarchical compression.
  • Encode/Decode Methods: Convert between raw data and quantized token indices.
# Example: Tokenize raw data
tokenizer = KronosTokenizer.from_pretrained("NeoQuasar/Kronos-Tokenizer-base")
x = torch.randn(1, 100, 6)  # (batch, seq_len, OHLCV+volume+amount)
indices = tokenizer.encode(x, half=True)  # Returns (s1_ids, s2_ids)

B. Kronos Model (Kronos)

A Transformer-based model that predicts future tokens using the hierarchical tokenization output. Key components:

  • HierarchicalEmbedding: Embeds s1 and s2 tokens into a shared feature space.
  • TemporalEmbedding: Adds time-based features (minute, hour, weekday) to the input.
  • DependencyAwareLayer: Ensures s2 tokens are conditioned on s1 tokens (critical for financial time series).
  • DualHead: Two separate heads for predicting s1 and s2 tokens.
# Example: Load pre-trained model
model = Kronos.from_pretrained("NeoQuasar/Kronos-small")
s1_logits, s2_logits = model(s1_ids, s2_ids, stamp=time_emb)

C. Predictor Wrapper (KronosPredictor)

A user-friendly API to simplify forecasting. Handles:

  • Data Normalization: Scales OHLCV data to a stable range.
  • Time Stamp Processing: Extracts temporal features (via calc_time_stamps).
  • Batch Prediction: Supports parallel forecasting for multiple assets.
  • Result Formatting: Returns pandas DataFrames with predicted OHLCV values.
# Example: Predict future prices
predictor = KronosPredictor(model, tokenizer)
pred_df = predictor.predict(
    df=historical_data,
    x_timestamp=historical_timestamps,
    y_timestamp=future_timestamps,
    pred_len=120  # Predict next 120 time steps
)

2. Inference Pipeline

Kronos uses autoregressive sampling to generate future tokens:

  1. Tokenization: Convert historical data to s1/s2 tokens.
  2. Autoregressive Generation: For each future step:
    • Predict s1 tokens using the current context.
    • Condition s2 prediction on the generated s1 tokens.
    • Use top_k/top_p filtering to control sampling diversity.
  3. Decoding: Convert predicted tokens back to raw OHLCV values.

Key inference functions:

  • auto_regressive_inference: Main loop for generating future tokens.
  • sample_from_logits: Samples tokens using temperature, top-k, or top-p.
  • top_k_top_p_filtering: Filters low-probability tokens to improve stability.

3. Key Features

  • Hierarchical Tokenization: Captures multi-scale patterns in financial data.
  • Dependency-Aware Prediction: Ensures s2 tokens (fine-grained) depend on s1 tokens (coarse-grained).
  • Batch Processing: Efficiently forecasts multiple assets in parallel.
  • Temporal Features: Incorporates time-based context (e.g., market hours, weekdays).
  • Open-Source: Pre-trained models available on Hugging Face (Kronos-mini, Kronos-small, Kronos-base).

4. Usage Workflow

  1. Install Dependencies:
    pip install -r requirements.txt
  2. Load Pre-Trained Models:
    Use from_pretrained to load tokenizer and model from Hugging Face.
  3. Prepare Data:
    Ensure input DataFrames have open, high, low, close, volume, and amount columns.
  4. Generate Predictions:
    Use KronosPredictor.predict or predict_batch for single/multiple assets.

5. Codebase Structure

The repo is organized into core modules:

  • model/kronos.py: Contains KronosTokenizer, Kronos, and inference functions.
  • model/module.py: Implements Transformer blocks, BSQuantizer, HierarchicalEmbedding, and utility layers.
  • predict.py: Wraps the predictor for command-line use.
  • examples/: Contains Jupyter notebooks and scripts for forecasting.

Kronos is designed for real-world financial applications—it balances performance with usability, making it accessible for both researchers and practitioners.

Understanding Traceroute (16 minute read)

Tech Deep Dive: How Traceroute Works (And Building It in Rust)

By Stonecharioteer (adapted for newsletter)

Traceroute isn’t magic—it’s a clever TTL (Time To Live) trick, and you can build a basic version in ~80 lines of Rust. Here’s the breakdown:

Core Mechanism

Traceroute sends UDP packets to a target (e.g., 8.8.8.8) with incrementing TTL values:

  1. Routers along the path decrement TTL by 1.
  2. When TTL hits 0, routers drop the packet and send an ICMP Time Exceeded (Type11) reply with their IP.
  3. The target sends an ICMP Port Unreachable (Type3) reply (traceroute uses unused port 33434) when TTL doesn’t expire.

Rust Implementation Highlights

  • Two Sockets: UDP (send probes with set TTL) + raw ICMP (listen for replies—requires sudo).
  • Packet Parsing: Reads raw IP headers (source IP at bytes12-15) and ICMP type (byte20) to distinguish hops from destination.
  • Enhancements: Added RTT timing (using Instant), 3 probes per TTL (variance/reliability), and stops at the target.

Key Limitations

  • Asymmetric Paths: Reply routes may differ from forward routes (traceroute doesn’t show this).
  • Missing Features: No DNS reverse lookup, port incrementing, or IPv6 support (real traceroute has these).

  4. Algorithm Upgrade: Kahn’s algorithm (O(V+E)) + BFS closure (process only needed flags) replaced O(n²) dependency checks.

  • p99 latency dropped from ~460ms → 94ms (spikes now ≤200ms).
  • Connection pool utilization fell to 10% (from 100% spikes).

OpenAI Academy: AI Fundamentals Summary

This beginner-friendly guide breaks down core AI concepts without requiring technical expertise:

What is AI?
AI refers to software that recognizes patterns, learns from data, and generates useful outputs—seen in everyday tools like map rerouting, fraud detection, and chatbots. It’s a broad category of specialized systems called models (e.g., speech, vision, forecasting).

Large Language Models (LLMs)
LLMs (powering tools like ChatGPT) focus on language: they predict the next logical text based on context (not "knowing" like humans). Labs like OpenAI build these models, then offer them via products (ChatGPT) or APIs for developers to integrate into tools.

Model Training Stages

  • Pre-training: Models learn general patterns from massive text (analogous to a new employee reading manuals).
  • Post-training: Models are coached to follow instructions, improve safety (reduce harmful outputs), and match tone (like a manager guiding an employee).

Model Types

  • Non-reasoning (Instant): Fast, fluent for simple tasks (drafting, summarizing).
  • Reasoning (Thinking): Slower, better at complex, step-by-step work (planning, debugging). ChatGPT defaults to auto-switching between these.

Key Hierarchy
AI → Models → LLMs → ChatGPT (product using LLMs).

For consistent results, be explicit about your goal, audience, and constraints.
